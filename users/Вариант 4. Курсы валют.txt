Вариант 4: Курсы валют
Цель
Создать мобильное Android-приложение для отображения текущего курса валют и исторических значений за предыдущие дни. Пользователь сможет отслеживать динамику курсов и сравнивать разные валюты.
Описание приложения
Приложение отображает список валют с текущим курсом, а также исторические данные за несколько предыдущих дней. Валюты визуализируются в виде сетки с использованием CardView или Compose Card. В дальнейшем приложение расширяется за счёт локального хранения через Room, работы с сервером, архитектуры MVVM, DI, графиков исторических данных и push-уведомлений о значительных изменениях курсов.



Часть 2 – Фрагменты и локальное сохранение
Функциональные требования
   1. Разделить приложение минимум на два фрагмента:
   * список валют с текущим курсом;
   * исторические курсы выбранной валюты.
   2. Реализовать навигацию между фрагментами.
   3. Передавать данные между фрагментами через Bundle или SafeArgs (если используется Navigation Component)
   4. Реализовать локальное сохранение списка валют между перезапусками приложения в SharedPreferences.
   5. Добавить возможность отмечать выбранные валюты в “избранное” и отображать их вверху экрана ДО обычных в сетке.
   * статус “избранного” также нужно сохранять в SharedPreferences.
   6. Использовать Snackbar для подтверждения действий:
   * при выборе валюты;
   * при обновлении исторических данных.


Часть 3 – Работа с сервером и асинхронность
Функциональные требования
      1. Получать данные о текущем курсе и истории с сервера через HTTP в формате JSON.
(URL https://api-labs.wiremockapi.cloud/exchange-rates
Альтернативный URL: https://gateway-staging.superglue.games/android/api/v1/public/mocks/exchange-rates).
      2. Асинхронные запросы реализовать с использованием:
         * Retrofit + Kotlin Coroutines.
         3. Отображать загруженные данные в интерфейсе.
         4. Реализовать автоматическое обновление данных одним из вариантов:
         * Раз в минуту через: Kotlin Coroutines + delay
         * Обновление по кнопке «Обновить»
         * Или через SwipeRefreshLayout
         5. Обрабатывать корректные и некорректные HTTP-ответы:
         * при ошибке отображать Snackbar.
         6. Каждая валюта должна иметь уникальный id (UUID или строка).
         7. Сетевые запросы не должны выполняться на главном потоке.
         8. Загруженные валюты и курсы объединять с локальными, обновляя их по id.


Часть 4 – Room, MVVM и Dependency Injection
Функциональные требования
            1. Использовать Room для кэширования текущего и исторических курсов валют.
            2. Приложение должно быть реализовано по архитектуре MVVM:
            * ViewModel управляет состоянием данных;
            * UI подписывается на изменения через LiveData или Kotlin Flow.
            * ViewModel не должна содержать ссылок на View или Context
            3. Настроить Dependency Injection:
            * Hilt или Koin для репозиториев и базы данных.
            4. Реализовать дополнительные функции:
            * фильтры по валютам;
            * сортировка по дате или курсу;
            * возможность быстро переключаться между валютами.
